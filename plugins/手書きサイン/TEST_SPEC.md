## 2. テスト目的

手書きサインプラグイン_testの設定画面、PC画面、モバイル画面、権限差、エラー時動作、BUG.mdに修正済み内容がある場合はその確認を含め、kintone実機で問題なく動作することを確認する。

プラグイン概要:

[PluginAdaptiX] v2.0.11 編集画面で手書きでサインをした内容を画像ファイルとして添付ファイルフィールドに保存することが出来るプラグインです。

## 3. テスト前提条件

| 項目 | 内容 |
|---|---|
| kintone環境 | テスト用kintone環境 |
| 使用アプリ | 手書きサインプラグイン_testを適用するテスト用アプリ |
| 使用ユーザー | 管理者ユーザー / 一般ユーザー |

| 必須設定キー | settings |

### テスト用アプリ・フィールド準備

- 管理者ユーザーでテスト用アプリを開く。
- プラグイン設定画面で選択できるフィールドを用意する。
- 文字列、数値、日付、ユーザー選択、サブテーブル、添付ファイルなど、プラグイン設定画面で必要になるフィールドを追加する。
- 既存データ確認用に、最低3件のレコードを登録する。
- モバイル確認を行う場合は、スマートフォンまたはChromeのモバイル表示を使用する。

### 設定画面で確認する主な項目

- 手書きサインを表示するスペース
- サインを保存する添付ファイル
- サインをした日時を格納するフィールド
- キャンセル
- 保存

### 使用フィールド・保存設定の確認メモ

- settings: space, file, date の配列
- `kintone.plugin.app.getConfig` による取得を確認した。
- `kintone.plugin.app.setConfig` による保存を確認した。
- 配列/オブジェクトはJSON文字列として保存されるものが多い。
- `settings`
- `kintone.plugin.app.getConfig()` による取得を確認した。
- `kintone.plugin.app.setConfig()` による保存を確認した。
- 配列や複数行設定はJSON文字列として保存される実装が多い。詳細な値構造は上記保存キー・保存値を参照。

## 11. 作成時の注意

- 素人でも操作できる言葉で書く
- 省略しない
- 「確認する」だけで終わらせず、何を見ればOKか書く
- プラグイン固有の機能に合わせて項目を追加する
- BUG.mdにバグがある場合は、バグ修正確認テストを必ず追加する
- 不明な場合は「未確認」と記載する
- 推測で断定しない
