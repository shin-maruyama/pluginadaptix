# Decisions

重要な設計判断を記録するディレクトリです。

## 目的

- なぜその設計や運用を選んだのかを後から追跡できるようにする。
- API、データモデル、認証、外部連携、運用方針の変更理由を明確にする。
- 実装だけでは読み取れない背景やトレードオフを残す。

## 記録対象

- API 仕様やエンドポイント構成の判断
- kintone アプリ定義、フィールド定義、データモデルの判断
- ライセンス認証、認可、セキュリティ方針の判断
- WordPress や外部サービスとの連携方式の判断
- 技術選定、運用方針、検証方針の変更

## ファイル命名

```text
YYYY-MM-DD-short-title.md
```

例:

```text
2026-06-20-api-auth-policy.md
```

## テンプレート

```md
# タイトル

- 日付:
- ステータス: proposed | accepted | superseded

## 背景

## 選択肢

## 決定

## 理由

## 影響範囲

## 関連リンク
```
