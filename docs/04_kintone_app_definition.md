APIサーバーはkintone REST APIを利用して各アプリへアクセスする。
この設計で進めると、次の工程として ER図（テーブル関連図）→ GitHubリポジトリ構成 → TypeScript(Node.js)認証API実装 → kintoneプラグイン実装 まで一気に落とし込めます。特に今回の構成は kintone をDB代わりに使う SaaS型ライセンス管理システムなので、ER図を先に確定させると Codex にそのまま実装させやすくなります。
